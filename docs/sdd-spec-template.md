## 3. 业务闭环定义

### 3.1 正向主流程

用文字或列表描述从上游输入到下游结果的主链路。

```text
上游来源
  -> MES 校验或转换
  -> 后台管理动作
  -> 现场终端或外部系统回执
  -> 异常或放行结果
  -> 追溯 / 报表 / 下游单据
```

示例：

```text
ERP 生产订单
  -> MES 主数据校验
  -> APS 排程
  -> 派工单下达
  -> 工位终端模拟扫码开工回执
  -> 过程记录与质量放行
  -> 工序报工、入库和追溯档案
```

## 7. UI 与视觉约束

- 使用中文 MES 业务语言，避免营销化表达。
- 页面信息密度应符合后台管理系统，优先清晰、克制、可扫描。
- 不使用单一色系堆叠，不新增无业务意义的装饰元素。
- 沿用现有 CSS 变量、面板、按钮、网格、状态类命名风格。
- 关注固定宽度、最小宽度、滚动区域和文本溢出，避免重叠和不可读。
- 主内容区优先使用左右独立纵向列，避免 CSS Grid 行高互相牵连造成大面积空白。
- 如果依赖 `hidden` 属性控制显示隐藏，CSS 必须保留：

```css
[hidden] {
  display: none !important;
}
```

## 8. 实现约束

### 8.1 静态项目边界

- 默认只编辑 HTML、CSS、JavaScript 和 Markdown。
- 使用原生 JavaScript，不引入框架。
- 不新增运行时依赖、构建步骤或后端服务。
- 页面可以直接打开，也可以通过静态文件服务预览。

### 8.2 文件组织

| 文件 | 责任 |
| --- | --- |
| `*.html` | 页面结构、语义容器、资源装配 |
| `*.css` | 视觉、布局、响应式、状态样式 |
| `*.js` | 演示数据、localStorage、事件绑定、DOM 渲染 |
| `UI_Design/data/*.js` | 跨页面共享演示数据或业务事实 |
| `UI_Design/navigation.js` | 菜单与页面入口 |
| `UI_Design/README.md` | 页面范围与打开方式说明 |

### 8.3 推荐新增文件

复杂功能应按业务域新增独立页面或模块，不把完整业务功能塞回首页。

```text
UI_Design/<business-domain>/<feature>.html
UI_Design/<business-domain>/<feature>.css
UI_Design/<business-domain>/<feature>.js
```

## 10. 验收标准

### 10.1 业务验收

- 页面能回答“谁在什么岗位、使用什么终端、处理哪张单、执行哪个动作、产生什么业务记录”。
- 能体现状态来源、操作校验、责任人、时间戳、单据或批次关联、异常处理、交接或追溯结果。
- 后台管理动作、真实生产动作和静态演示中的模拟外部动作区分清楚。
- 与上游 ERP / PLM / WMS / QMS / SCADA / PLC 关系清晰，不越界。

### 10.2 功能验收

- 页面能正常加载，无控制台 JavaScript 错误。
- 搜索、筛选、选择、提交、关闭、重新打开、重置演示等交互可用。
- 修改后的演示状态能写入 localStorage，刷新后保留。
- 点击“重置演示”后，数据和演示状态回到初始值。
- 空状态、异常状态、校验失败状态有明确反馈。

### 10.3 视觉验收

- 主要区域无明显遮挡、溢出或不可读文本。
- 侧边栏、顶部操作区、主内容区和面板内部密度协调。
- 桌面和常见移动视口下无控件重叠。
- 抽屉、弹窗、悬浮窗、右侧详情面板有关闭和重新打开入口。
