| 项目 | 内容 |

## 2. 背景与目标

MES 需要兼容两类工厂：

- 有 ERP/PLM/SRM：订单、产品、物料、BOM、工艺、客户、供应商来自外部系统。
- 无 ERP/PLM/SRM：上述对象允许在 MES 内手工维护。

本批目标是把订单与基础资料从“外部系统依赖”调整为“双来源受控维护”，避免普通 CRUD。

## 3. 订单与计划整改

| 页面 | 当前风险 | 整改要求 |
| --- | --- | --- |
| 生产订单 | “新增订单”“删除/作废订单”像无约束 CRUD | 保留新增，改为“新建 MES 手工工单”；增加来源类型；草稿可删除，已引用只能作废、暂停或关闭 |
| 订单评审 | 需兼容无 ERP/PLM 场景 | 评审对象既可来自 ERP 工单，也可来自 MES 手工工单；校验产品、BOM、工艺、物料、客户要求、检验规则 |
| 生产排程 | “确认排程”可能影响已下达任务 | 输出排程版本；对已下达或生产中任务必须走计划调整或任务变更 |
| 产能负荷 | “生成均衡建议”不能直接改变排程 | 保持建议性质，写入计划调整候选方案 |
| 交期预警 | “设为加急”可能绕过优先级审批 | 改为“提交加急会签/调整优先级申请” |
| 计划调整 | “方案提前/后移/拆批”不能直接生效 | 改为调整草案，审批和影响校验通过后生成排程版本 |
| 齐套检查 | “锁定库存”“催采购”边界需兼容有无 WMS/采购系统 | 有 WMS 时生成锁库请求并接收回执；无 WMS 时登记 MES 库存保留记录；催采购改为登记协同请求 |

## 4. 基础资料整改

| 页面 | 整改要求 |
| --- | --- |
| 产品资料 | 支持 PLM 同步和 MES 手工维护；手工版本必须审批、签收、生效和引用检查 |
| 物料资料 | 支持 ERP 同步和 MES 手工维护；增加批次规则、IQC、替代料和投料防错 |
| BOM 清单 | 支持 MES 手工 BOM；必须有版本、用量、损耗、替代料、审批和生效范围 |
| 工艺路线 | 支持 MES 手工工艺；必须有工序、工位、设备、SOP、检验点和签收 |
| 工序工位 | 启用需校验设备、终端、人员资质、数据采集和开工准入 |
| 产线车间 | 维护资源日历、产能、班次、瓶颈资源和例外窗口 |
| 规则与代码 | 管理不良代码、停机原因、批次规则、标签规则、抽样规则 |
| 班次日历 | 作为排程、派工、OEE 和交接班统一时间口径 |
| 客户供应商 | 客户维护标签、包装、OQC、追溯报告和交付要求；供应商维护可供物料、质量等级、IQC、批次规则；不维护价格、合同、账期、财务 |

## 5. 手工维护对象必填字段

| 对象 | 必填字段 |
| --- | --- |
| 生产订单 | 工单号、产品、数量、交期、客户、来源类型、评审状态、创建人、责任计划员 |
| 客户 | 客户编码、客户名称、客户料号规则、标签模板、包装要求、OQC 要求、追溯报告要求 |
| 供应商 | 供应商编码、供应物料范围、质量等级、IQC 策略、批次规则、状态、责任人 |
| 产品 | 产品编码、版本、状态、BOM 引用、工艺引用、检验规范、标签模板、生效范围 |
| 物料 | 物料编码、批次规则、IQC 策略、替代料、供应商范围、投料防错要求 |
| BOM | BOM 编号、版本、子项物料、用量、损耗率、使用工序、替代料、生效范围 |
| 工艺路线 | 路线编号、版本、工序、工位、设备、SOP、检验点、首件触发条件 |

## 6. 状态与引用约束

| 场景 | 规则 |
| --- | --- |
| 草稿未引用 | 允许编辑和删除，但必须记录创建/删除人和时间 |
| 已评审 | 允许变更申请，不允许无痕改关键字段 |
| 已排程 | 修改数量、交期、BOM、工艺需触发计划影响评估 |
| 已派工 | 修改任务、工位、SOP、物料需触发任务变更 |
| 已报工/入库 | 不允许改原始事实，只能做冲销、补录、复核或差异调整 |
| 已追溯引用 | 任何改动必须保留版本和新旧关系 |

## 7. 验收标准

- 订单、客户、供应商、产品、物料、BOM、工艺支持“外部同步 / MES 手工维护”两种来源表达。
- 手工维护对象具备版本、状态、责任人、时间戳、复核/审批、引用影响和审计履历。
- 已被订单、派工、报工、库存或追溯引用的数据不可无痕删除。
- 无外部系统场景下，订单评审、排程、齐套和基础资料仍能形成闭环。
