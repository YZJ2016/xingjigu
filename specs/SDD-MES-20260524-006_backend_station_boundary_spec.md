## 2. 背景与目标

### 2.1 当前系统现状

- `工位作业` 作为一级菜单存在，包含员工登录、扫码开工、投料确认、过程记录、工序报工等现场终端动作。
- 项目要求后台页面不能替代真实现场终端执行生产动作；模拟扫码、HMI、PDA、工牌/NFC、PLC 等必须明确出现“模拟”。
- `报表与看板 / 管理驾驶舱` 容易越界到 BI 或经营综合大屏。
- `基础资料 / 客户供应商` 容易越界到 CRM/SRM 主数据维护。

### 2.2 业务目标

- 让后台用户明确知道：当前页面是在查看、配置、复核、补偿或审计现场动作，而不是直接执行真实生产动作。
- 保留现场终端演示价值，但所有模拟控件必须带有来源、时间、回执、责任人和追溯记录。
- 报表与基础资料页面保持 MES 范围，不扩展成 BI、CRM 或 SRM。

### 2.3 非目标

- 不删除 `工位作业` 一级菜单。
- 不接入真实扫码枪、PDA、HMI、PLC、SCADA、WMS 或 ERP。
- 不把报表驾驶舱扩展为企业经营 BI。
- 不把客户供应商维护扩展为 CRM/SRM 全功能系统。

## 3. 业务闭环定义

### 3.1 工位作业边界

```text
后台派工 / 准入规则
  -> 工位终端展示任务
  -> 模拟扫码枪 / HMI / PDA / 工牌/NFC 回执
  -> MES 后台记录校验、拦截、补偿、追溯
  -> 质量、物料、设备、异常、完工入库继续闭环
```

### 3.2 角色与动作边界

| 页面类别 | 后台可以做 | 后台不应表达为直接做 |
| --- | --- | --- |
| 员工登录 | 配置登录准入、查看模拟工牌/NFC 回执、锁定异常登录 | 真实刷工牌、替员工登录 |
| 扫码开工 | 查看模拟扫码回执、开工准入拦截、异常转单 | 真实扫码、真实启动设备 |
| 投料确认 | 查看模拟投料回执、批次拦截、差异复核 | 真实投料、真实扣料 |
| 工序报工 | 记录模拟 HMI/PDA 报工回执、报工复核 | 真实报工、真实 ERP 入账 |
| 过程监控 | 监控 PLC/SCADA/API 回传、标记异常、派单 | 真实控制设备启停 |
| 管理驾驶舱 | 发布 MES 可信生产指标、下钻到 MES 事实 | 替代 BI 做跨财务经营分析 |
| 客户供应商 | 维护 MES 引用范围、检验策略、追溯规则 | 维护完整 CRM/SRM 客商档案 |

## 4. 页面范围

### 4.1 工位作业页面

- 菜单层建议将 `工位作业` 的 `layer` 从“现场操作”统一为“现场终端”或“现场回执”。
- 页面副标题必须包含“模拟现场回执”或“工位终端回执”。
- 所有按钮若触发外部动作演示，必须包含“模拟”。

### 4.2 报表与看板页面

- `管理驾驶舱` 定位为 `MES 生产指标驾驶舱`。
- 指标来源限定为 MES、ERP 工单、WMS 入库、质量放行、设备状态、异常闭环。
- 不新增财务、采购、销售利润等非 MES 指标。

### 4.3 基础资料页面

- `客户供应商` 页面只维护 MES 执行引用信息。
- 供应商侧重点：资质状态、质量等级、来料检验策略、批次追溯规则。
- 客户侧重点：标签模板、追溯报告要求、交付优先级引用。

## 5. 数据规格

### 5.1 模拟回执统一字段

```js
{
  id: "",
  sourceType: "模拟扫码枪 | 模拟PDA | 模拟HMI | 模拟PLC | 模拟WMS | 模拟ERP",
  sourceDevice: "",
  businessObject: "",
  orderId: "",
  dispatchId: "",
  station: "",
  operator: "",
  receivedAt: "",
  validationResult: "",
  blockedReason: "",
  traceRef: ""
}
```

### 5.2 必填业务字段

| 字段 | 说明 |
| --- | --- |
| sourceType | 必须明确是“模拟”来源 |
| businessObject | 对应工单、派工、批次、SN、设备或单据 |
| validationResult | 后台校验结果 |
| traceRef | 写入追溯或履历的引用 |

## 6. 验收标准

- 工位作业全组页面的模拟动作标题、按钮或占位提示均出现“模拟”。
- 页面不出现“后台启动设备”“后台真实投料”“后台真实扫码”等误导性表达。
- 管理驾驶舱文案限定为 MES 生产指标，不扩展为综合 BI。
- 客户供应商页面不承担库存、采购、销售、合同、价格维护。
- 所有模拟动作写入 localStorage 或 `MES_BUSINESS_FLOW` 履历。

## 7. 实施顺序

1. 统一 `工位作业` 菜单层级文案。
2. 扫描并修正工位作业按钮、标题、占位提示。
3. 收敛管理驾驶舱指标说明。
4. 收敛客户供应商页面职责说明。
5. 逐页验证“模拟”文案和业务履历。
