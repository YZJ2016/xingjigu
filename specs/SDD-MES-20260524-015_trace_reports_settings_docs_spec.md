## 2. 业务边界

- 追溯查询生成快照和影响范围，不扩展成 CRM 或客户服务系统。
- 报表与看板只展示 MES 已校验生产事实，不改原始业务记录。
- 系统设置服务 MES 治理，不替代审批人、接口系统、IT 运维或外部账务。
- README 需说明有外部系统和无外部系统两种模式。

## 3. 追溯查询整改

| 页面 | 整改要求 |
| --- | --- |
| 产品追溯 | 按 SN 或成品批次反查工单、物料、工序、检验、设备、包装和客户 |
| 批次追溯 | 支持原料批次和成品批次正反向追溯 |
| 物料去向 | 串联出库、线边签收、投料、余料和成品去向 |
| 生产履历 | 汇聚扫码、HMI、PLC、报工和交接记录 |
| 检验履历 | 串联 IQC、FAI、IPQC、FQC、NCR/MRB 和放行 |
| 设备履历 | 关联设备状态、点检、维修、停机、校准和生产批次 |
| 客户追溯报告 | 只生成质量追溯报告快照，不扩展成 CRM 或客户服务系统 |

## 4. 报表与看板整改

| 页面 | 整改要求 |
| --- | --- |
| 生产日报 | 只发布快照，不改原始报工、检验、入库和设备事实 |
| 良率分析 | 关联 NCR/MRB、返工和质量闭环 |
| 交付达成 | 有外部系统时引用 ERP/WMS；无外部系统时引用 MES 手工订单、MES 入库和交付计划 |
| 设备效率报表 | 区分于设备模块工作台，只展示报表快照和下钻 |
| 停机损失 | 来源 PLC/SCADA/班组补录/维修归因 |
| 物料损耗 | 来源 BOM、投料、报工核销、余料退回和库存记录 |
| MES 生产指标驾驶舱 | 仅展示 MES 已校验生产事实；“发布会议版本”改为“锁定驾驶舱快照” |

## 5. 系统设置整改

| 页面 | 整改要求 |
| --- | --- |
| 人员账号 | 和工位身份回执区分，后台管账号、范围、状态、锁定 |
| 人员资质 | 支撑开工准入、质量签核、维修验收和电子签名 |
| 角色权限 | 联动菜单、按钮、数据范围、电子签名和审计 |
| 审批设置 | 配置流程，不替代审批人签核 |
| 接口补偿配置 | 只配置重试、补偿、审批、对账规则，不直接改外部账 |
| 消息提醒 | 提醒不等于异常关闭 |
| 操作记录 | 不允许无痕删除或篡改 |
| 数据备份 | 可保留在系统设置，但改为“数据归档与恢复演练”，只展示策略、演练、告警和责任闭环 |

## 6. README 整改

`UI_Design/README.md` 需补充：

- 项目支持有外部系统和无外部系统两种 MES 原型模式。
- 无外部系统时，MES 可承担受控手工维护，但必须有来源、版本、复核/审批、引用影响和审计。
- 后台页面不替代现场终端真实动作，所有外部或现场动作在静态原型中均为模拟回执。
- 报表和驾驶舱只展示 MES 已校验生产事实，不作为 BI、财务、销售、采购综合分析系统。

## 7. 验收标准

- 追溯页面能解释查询快照、影响范围、关联事实和追溯引用。
- 报表页面不改原始事实，只做快照、复核、锁定、下钻和改善项。
- 系统设置页面不直接替代审批、外部账务或 IT 生产数据库操作。
- README 明确双模式和现场边界。
