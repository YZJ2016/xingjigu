## 2. 背景与目标

当前多个业务 JS 内仍保留旧版 `modules` 或 `pageMaps` 硬编码菜单，和 `navigation.js` 不一致，导致菜单项、命名、路由和业务边界可能漂移。

目标：

- 所有页面统一使用 `UI_Design/navigation.js` 的 `MES_NAV_MODULES` 和 `MES_NAV_ROUTES`。
- 修正首页偏演示化按钮文案。
- 建立全局按钮和模拟动作命名规则。

## 3. 整改范围

### 3.1 统一导航数据源

| 问题 | 整改要求 |
| --- | --- |
| 各模块 JS 重复定义菜单 | 删除或停用重复菜单，统一读取 `window.MES_NAV_MODULES` |
| 旧菜单项不一致 | 对齐 `navigation.js`：`接口补偿配置`、`停机归因`、`工序完工复核`、`人员资质`、`规则与代码`、`班次日历`、`出货检验` |
| 页面内手写跳转逻辑过多 | 优先使用 `resolveMesRoute(moduleId, entry)` |

### 3.2 建议菜单命名

| 当前菜单 | 建议命名 | 原因 |
| --- | --- | --- |
| 员工登录 | 工位身份回执 | 避免像普通后台登录 |
| SOP 查看 | 工艺文件与作业指导 | 覆盖 SOP 版本、下发、签收、变更 |
| 管理驾驶舱 | MES 生产指标驾驶舱 | 避免泛 BI 化 |
| 数据备份 | 数据归档与恢复演练 | 避免偏离生产业务主链路 |

### 3.3 首页工作台

| 当前要素 | 整改要求 |
| --- | --- |
| 新建异常 | 改为“登记后台异常补录” |
| 切换繁忙产线 | 改为“按产线筛选” |
| 刷新数据 | 改为“接收模拟接口刷新回执” |
| 重置演示 | 保留，但作为原型工具弱化，不混入业务处置区 |

## 5. 验收标准

- 页面 JS 中不再出现与 `navigation.js` 内容不一致的菜单副本。
- 所有页面导航项和路由一致，左侧菜单可达。
- 首页按钮不表达为演示玩具动作。
- 全量页面加载无导航相关 JavaScript 错误。
