## 2. 背景与目标

### 2.1 当前系统现状

- 设备模块覆盖设备状态、点检、保养、维修、停机、备件、设备效率。
- 质量模块覆盖 IQC、首件、巡检、过程、FQC、不良、返工、质量放行。
- 异常模块覆盖异常上报、待处理、停线、缺料、质量问题、设备故障、处理复盘。
- 当前缺少工装夹具、量检具校准、OQC 出货检验、CAPA/8D 改善的明确入口或页面职责。

### 2.2 业务目标

- 将工装夹具、测试治具、量检具和校准状态纳入开工准入、过程检验和追溯。
- 根据企业业务需要补充 OQC 出货检验，区分 FQC 成品放行和客户出货前检验。
- 将重大或重复异常从“处理复盘”升级为可闭环的 CAPA/8D 改善对象。

### 2.3 非目标

- 不替代 PLM 的工装设计和图纸管理。
- 不替代 QMS 的完整质量体系文档管理。
- 不替代实验室 LIMS 或真实校准系统。
- 不强制所有企业都启用 OQC；如果 FQC 已覆盖出货检验，可配置为不启用独立 OQC。

## 4. 业务闭环定义

### 4.1 工装与校准闭环

```text
基础资料维护工装/量检具
  -> 派工和开工检查引用有效状态
  -> 现场模拟工装/检具回执
  -> 过程检验和质量放行引用校准证据
  -> 追溯记录保留使用版本、有效期和责任人
```

### 4.2 OQC 闭环

```text
质量放行 / 成品入库 / 出货批次
  -> OQC 抽样或客户检验要求
  -> 模拟检验回执
  -> 出货放行 / 拦截 / 客户报告
  -> 追溯查询和客户追溯报告
```

### 4.3 CAPA 闭环

```text
重大或重复异常 / NCR / 客诉
  -> RCA 原因分析
  -> 纠正措施和预防措施
  -> 责任人、期限、验证计划
  -> 有效性验证
  -> 关闭或升级复盘
```

## 6. 页面范围

| 页面 | 首屏业务重点 | 下游影响 |
| --- | --- | --- |
| 工装夹具 | 工装状态矩阵、绑定工序、寿命/校准风险 | 开工检查、过程检验、追溯 |
| 量检具校准 | 校准到期队列、检验设备有效性 | 首件、过程检验、FQC、OQC |
| 出货检验 | 出货批次、客户要求、抽样结果、标签复核 | 客户追溯报告、发货放行 |
| CAPA 改善 | RCA、措施、责任人、期限、验证结果 | 异常复盘、质量改善、报表 |

## 7. 交互规格

| 动作 | 前置条件 | 校验规则 | 成功结果 | 失败结果 |
| --- | --- | --- | --- | --- |
| 工装状态复核 | 工装已建档 | 工序、工位、有效期必填 | 状态更新，开工准入读取 | 阻止可用状态 |
| 登记模拟校准回执 | 校准任务存在 | 证书号、有效期、责任人必填 | 校准状态更新，追溯可引用 | 生成校准异常 |
| OQC 放行 | 抽检完成 | 客户要求、标签、FQC 放行、冻结状态通过 | 出货检验合格，客户报告可引用 | 拦截出货批次 |
| CAPA 关闭 | 措施已验证 | RCA、措施、证据、验证结论必填 | 状态变为已关闭 | 逾期升级或退回 |

## 8. 验收标准

- 工装或校准过期能在开工检查、过程检验或质量放行中体现为准入风险。
- OQC 如果启用，必须与 FQC、质量放行和客户追溯报告形成清晰关系。
- CAPA 页面不只是复盘文字，应包含 RCA、措施、责任人、期限和有效性验证。
- 所有模拟校准、检验、验证动作写入履历，并包含来源、责任人、时间戳和追溯引用。
- 不引入新依赖，不接入真实 QMS/LIMS。

## 9. 实施顺序

1. 补工装夹具和量检具基础数据。
2. 在开工检查和质量检验页面展示工装/校准引用。
3. 根据业务开关新增或扩展出货检验。
4. 将处理复盘扩展为 CAPA 改善闭环。
5. 补充追溯和报表引用。
