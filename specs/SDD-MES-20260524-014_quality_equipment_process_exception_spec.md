## 2. 业务边界

- 质量动作必须由质量角色、检验任务、检验项、签核和追溯引用支撑。
- 设备页面不直接控制设备启停，不直接完成维修，只记录状态、派工、验收和复核。
- 过程监控不改原始 PLC/SCADA/HMI/测试台数据，只聚合、预警、派单和归因。
- 异常处理负责分级、派单、处置、验证和复盘，不替代现场动作。

## 3. 质量检验整改

| 页面 | 整改要求 |
| --- | --- |
| 来料检验 | 有外部系统接收回执；无外部系统由 MES 登记 IQC 判定、冻结/放行和批次准入 |
| 首件检验 | 首件签核必须带质量角色、检验项、签名和放行/拦截结果 |
| 巡检任务 | 回执来源为模拟 PDA/工位终端/质量员录入 |
| 过程检验 | 测试台、PLC、SPC 来源必须保留，不直接改原始参数 |
| 成品检验 | FQC 放行需质量角色签核，不由普通按钮直接完成 |
| 质量放行 | 后台记录签核、拦截和追溯引用，不替代现场检验 |
| 出货检验 | 定位发运前质量门，不扩展为物流发货 |
| 不良记录 | 必须绑定 SN/批次/工序/设备/人员/物料批次 |
| 返工评审 | 页面下发返工路线和复检要求，返工作业仍来自工位回执 |

## 4. 设备与保养整改

| 页面 | 整改要求 |
| --- | --- |
| 设备状态 | 只记录状态复核、异常派单和恢复验证 |
| 工装夹具 | 准入结果来自模拟点检/借用/绑定回执 |
| 量检具校准 | 校准结果需证书、有效期、责任人和适用范围 |
| 点检计划 | “确认点检回执”改为“接收模拟点检回执/复核漏检” |
| 保养计划 | 记录保养验收、恢复产能和计划影响 |
| 维修工单 | 维修执行来自维修人员或移动端回执，后台做派工、验收、复核 |
| 停机记录 | “确认停机归因”改为“复核停机归因”，来源 PLC/SCADA/班组补录/维修结论 |
| 备件领用 | 有 WMS 接收出库回执；无 WMS 登记 MES 备件出库和安装设备 |
| 设备效率 | “重算 OEE”改为“重算 OEE 口径快照”，保留原始状态、产量和质量来源 |

## 5. 过程监控整改

| 页面 | 整改要求 |
| --- | --- |
| 实时产量 | 产量来自 PLC/HMI/报工草稿，不直接报工 |
| 设备运行 | 不表达为控制设备启停 |
| 工艺参数 | 不直接编辑原始参数 |
| 报警记录 | 不直接制造或关闭现场报警 |
| 停机归因 | 班组补录必须标明模拟 HMI/人工复核 |
| 过程趋势 | 聚合时序数据，不改原始采集明细 |
| 电子看板 | 改为“电子看板发布与现场签收”，保留刷新 ACK 和追溯入口 |

## 6. 异常处理整改

| 页面 | 整改要求 |
| --- | --- |
| 异常上报 | 来源为工位、PDA、Andon、PLC、人工补录或模拟回执 |
| 待处理异常 | 响应需带 SLA、责任人和响应时间 |
| 停线申请 | 审批停线不等于后台启停产线；需记录锁定范围、恢复条件和排程影响 |
| 缺料处理 | 改为“登记缺料处置方案/接收 WMS、采购或 MES 库存回执” |
| 质量问题 | 必须由质量角色签核，联动 NCR/MRB、隔离、返工或放行 |
| 设备故障 | 维修验收来自维修工单闭环 |
| 处理复盘 | 聚焦 RCA、CAPA、TPM 改善和有效性验证 |

## 7. 验收标准

- 质量签核具备角色、检验项、签名、结果和追溯引用。
- 设备页面不表达为后台控制设备或直接完成维修。
- 过程监控页面不改原始采集数据。
- 异常页面具备来源、等级、责任人、SLA、处置、验证和复盘。
- 所有模拟采集和外部回执文案明确。
